# RX_DISP_LOT

> The RX_DISP_LOT table contains identification number of the lots from which a fill is dispensed.

**Primary key:** `ORDER_MED_ID`, `CONTACT_DATE_REAL`, `GROUP_LINE`, `VALUE_LINE`  
**Columns:** 5

[← index](../index.md)

## Columns

| # | Column | Type | Flags | Description |
|--:|--------|------|-------|-------------|
| 1 | `ORDER_MED_ID` | NUMERIC | PK FK→ | The unique identifier for the order record. |
| 2 | `CONTACT_DATE_REAL` | FLOAT | PK | A unique contact date in decimal format. The integer portion of the number indicates the date of contact. The digits after the decimal distinguish different contacts on the same date and are unique for each contact on that date. For example, .00 is the first/only contact, .01 is the second contact, etc. |
| 3 | `GROUP_LINE` | INTEGER | PK | The line number of the NDC lot number in the order's record. Together with ORDER_MED_ID, this forms the foreign key to the RX_DISP_LOT table. |
| 4 | `VALUE_LINE` | INTEGER | PK | The line number of one of the NDC lot number lines associated with the medication order from the RX_DISP_LOT table. |
| 5 | `CONTACT_DATE` | DATETIME |  | The date of this contact in calendar format. |
