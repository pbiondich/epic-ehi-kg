# CAREPLAN_INFO

> Contains information about care plan template records.

**Primary key:** `CARE_INTG_ID`  
**Columns:** 15  
**Joined by:** 7 columns

[← index](../index.md)

## Columns

| # | Column | Type | Flags | Description |
|--:|--------|------|-------|-------------|
| 1 | `CARE_INTG_ID` | VARCHAR | PK | The unique identifier for the care plan record. |
| 2 | `CAREPLAN_TYPE_C_NAME` | VARCHAR |  | The category ID of the type of the care plan record (Collaborative or Home Health). |
| 3 | `PAT_ENC_CSN_ID` | NUMERIC | FK→ | The linked unique contact serial number for the patient. This number is unique across all patient encounters in your system. If you use IntraConnect, this is the Unique Contact Identifier (UCI). This column is frequently used to link to the PAT_ENC_HSP table. |
| 4 | `PATIENT_ID` | VARCHAR |  | Links OP care plan (patient-level and episodic) to the associated patient |
| 5 | `LINKED_PAT_CAREPLAN_YN` | VARCHAR |  | Indicates whether the outpatient care plan is linked at the patient level or the episode level. Yes means it is patient level, No means it is episode level. |
| 6 | `RFL_INSTR_NOTE_ID` | VARCHAR |  | Stores the ID of the HNO record that contains the referral instructions |
| 7 | `READING_CAREPLAN_ID` | VARCHAR |  | The link to the care plan reading Care Plan record. |
| 8 | `LAST_EDITED_DTTM` | DATETIME (Local) |  | The date and time when the care plan was last edited. This does not include documentation-only changes. |
| 9 | `RECORD_STATUS_2_C_NAME` | VARCHAR |  | Record status flag. Used in conjunction with record archived flag for encounter archiving. |
| 10 | `PAT_ENROLL_DEPARTMENT_ID_EXTERNAL_NAME` | VARCHAR |  | The external name of the department record. This is often used in patient correspondence such as reminder letters. |
| 11 | `IS_AUTOGENERATED_YN` | VARCHAR |  | This item determines whether a care plan is autogenerated. |
| 12 | `PT_TSKS_STAT_C_NAME` | VARCHAR |  | The current status of patient-assigned tasks within the care plan. |
| 13 | `PT_TSKS_STAT_RSN_C_NAME` | VARCHAR |  | The reason for the current status of patient-assigned tasks within the care plan. |
| 14 | `PT_TSKS_START_LOCAL_DTTM` | DATETIME (Local) |  | The instant in local time of when patient-assigned tasks were first present in the care plan. For a Care Companion care plan, this is synonymous with care plan's start instant. |
| 15 | `PT_TSKS_END_LOCAL_DTTM` | DATETIME (Local) |  | The instant in local time of when patient-assigned tasks in the care plan were no longer active. For a Care Companion care plan, this is synonymous with care plan's end instant. |
