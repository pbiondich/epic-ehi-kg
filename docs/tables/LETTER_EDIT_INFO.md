# LETTER_EDIT_INFO

> The LETTER_EDIT_INFO table holds audit information for letters. This table contains information pertaining to routing and letter edits.

**Primary key:** `NOTE_CSN_ID`, `LINE`  
**Columns:** 3

[← index](../index.md)

## Columns

| # | Column | Type | Flags | Description |
|--:|--------|------|-------|-------------|
| 1 | `NOTE_CSN_ID` | NUMERIC | PK FK→ | The unique contact serial number (CSN) of the note contact. If you use IntraConnect, this is the Unique Contact Identifier (UCI). |
| 2 | `LINE` | INTEGER | PK | The line number for the information associated with this contact. Multiple pieces of information can be associated with this contact. |
| 3 | `LETTER_AUDIT_COMMENT` | VARCHAR |  | This field is used to store additional data about a particular audit action (where a letter was printed from, why a letter failed to print, etc...). |

_Flags: PK = primary key · org = may contain organization-specific values · discont. = discontinued · FK→ = inferred reference (see below) · shared = generic key, intentionally unresolved._
