# CMGMT_EPISODE_CASE_TEAM

> This table contains the providers and pools that are responsible for performing ongoing outreach related to a Coordinated Care Management episode.

**Primary key:** `EPISODE_ID`, `LINE`  
**Columns:** 14  
**Org-specific columns:** 1

[← index](../index.md)

## Columns

| # | Column | Type | Flags | Description |
|--:|--------|------|-------|-------------|
| 1 | `EPISODE_ID` | NUMERIC | PK FK→ | The unique identifier for the summary block record. |
| 2 | `LINE` | INTEGER | PK | The line number for the information associated with this record. Multiple pieces of information can be associated with this record. |
| 3 | `MANAGER_PROV_ID_PROV_NAME` | VARCHAR |  | The name of the service provider. This item may be hidden in a public view of the CLARITY_SER table. |
| 4 | `POOL_REGISTRY_ID` | NUMERIC |  | Stores the pool of providers responsible for performing ongoing patient outreach related to this Coordinated Care Management episode. |
| 5 | `POOL_REGISTRY_ID_REGISTRY_NAME` | VARCHAR |  | The name of the In Basket registry in the HIP master file. |
| 6 | `EFFECTIVE_DATE` | DATETIME |  | Stores the date on which the provider or pool starts being responsible for reviewing the Coordinated Care Management episode. |
| 7 | `TERMINATION_DATE` | DATETIME |  | Stores the date on which the provider or pool stops being responsible for reviewing the Coordinated Care Management episode. |
| 8 | `RELATIONSHIP_C_NAME` | VARCHAR | org | Stores the type of relationship between the case manager and patient. |
| 9 | `REV_FREQ_UNIT_C_NAME` | VARCHAR |  | Stores the review frequency unit that indicates how often the manager or pool performs patient outreach. |
| 10 | `REVIEW_DUE_DATE` | DATETIME |  | Stores the date by which the provider or pool must next complete the Coordinated Care Management episode review. |
| 11 | `REVIEW_DUE_TASK_ID` | VARCHAR |  | Stores the task record in the Coordinated Care Management episode checklist that indicates the next review date. This task is generated by the system when a review due date (I HSB 18140) is set. |
| 12 | `LAST_REVIEW_DATE` | DATETIME |  | Stores the date on which the provider or pool performed the last Coordinated Care Management episode review. |
| 13 | `MANAGER_IS_PRIMARY_YN` | VARCHAR |  | This flag indicates whether the provider or pool is primarily responsible for the Coordinated Care Management episode. Only one provider/pool can be primarily responsible on any given effective date. |
| 14 | `ASSIGN_MATCH_LEVEL` | VARCHAR |  | This item will be set when providers are assigned to this episode using a Workload Balancer. Stores the Match Level of the assignment made by the Workload Balancer. |

_Flags: PK = primary key · org = may contain organization-specific values · discont. = discontinued · FK→ = inferred reference (see below) · shared = generic key, intentionally unresolved._
