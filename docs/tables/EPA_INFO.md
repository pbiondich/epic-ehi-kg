# EPA_INFO

> This table holds information about electronic prior authorization requests.

**Overflow family:** [EPA_INFO_2](EPA_INFO_2.md)  
**Primary key:** `REFERRAL_ID`  
**Columns:** 12

[← index](../index.md)

## Columns

| # | Column | Type | Flags | Description |
|--:|--------|------|-------|-------------|
| 1 | `REFERRAL_ID` | NUMERIC | PK FK→ | The unique identifier for the referral record. |
| 2 | `EPA_REF_ID` | VARCHAR |  | This item holds the reference ID for electronic prior authorization. It is generated by the system and used for linking a referral record to an electronic prior authorization message. This value will not change during an electronic prior authorization interaction. |
| 3 | `PA_PAT_NOTIFIED_YN` | VARCHAR |  | This item specifies whether the patient has been notified about the status of the prior authorization. |
| 4 | `PA_APPROVAL_SRC_C_NAME` | VARCHAR |  | This item records the entry source of the outpatient prescription prior authorization details. If an authorization was completed using both electronic and manual entry, this will be based on how the most recent data was entered. |
| 5 | `MED_EPA_UPD_UTC_DTTM` | DATETIME (UTC) |  | This holds the instant that ePA information was last updated on this referral. |
| 6 | `EPA_RFL_ORD_ID` | NUMERIC |  | This virtual item holds the order ID associated with an outpatient prescription medication prior authorization request. |
| 7 | `PAT_EPA_ENC_CSN_ID` | NUMERIC | FK→ | This item holds the CSN of the encounter the prior authorization was requested in. |
| 8 | `EPA_PHR_ID` | NUMERIC |  | This item stores the pharmacy that initiated prior authorization. |
| 9 | `EPA_PHR_ID_PHARMACY_NAME` | VARCHAR |  | The name of the pharmacy. |
| 10 | `EPA_REQ_TYPE_C_NAME` | VARCHAR |  | This item records the type of the outpatient prescription prior authorization request. 1-Prospective is a request that was initiated at the time of signing the order. 2-Retrospective is a request that was initiated after the order was signed. 3-Retrospective-RxChange is a retrospective request that was initiated by an electronic RxChange message from the pharmacy. 4-Renewal is a request that was initiated on a signed order that already had a prior authorization request linked to it. 5-Renewal-RxChange is a renewal that was initiated by an electronic RxChange message from the pharmacy. 6-Retrospective-Willow Ambulatory is a retrospective request that was initiated from within the Willow Ambulatory application. 7-Renewal-Willow Ambulatory is a renewal that was initiated from within the Willow Ambulatory application. |
| 11 | `CUR_EPA_STATUS_C_NAME` | VARCHAR |  | This item holds the current status of the outpatient prescription prior authorization request. |
| 12 | `EPA_HRS_FINAL_STAT` | INTEGER |  | This column holds the numbers of hours that the outpatient prescription prior authorization request was in progress. For requests that were initiated, but no other actions have taken place, the count returned will be 0. For other requests, the count returned will be the hours between the initiation of the request and the last action performed on the request. |

_Flags: PK = primary key · org = may contain organization-specific values · discont. = discontinued · FK→ = inferred reference (see below) · shared = generic key, intentionally unresolved._
